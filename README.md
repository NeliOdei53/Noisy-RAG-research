# Исследование устойчивости RAG-систем к шуму в базе знаний

Проект посвящён исследованию устойчивости Retrieval-Augmented Generation систем к различным видам шума в базе знаний.  
В рамках работы реализован полный экспериментальный пайплайн: от построения векторной базы знаний и генерации шума до запуска controlled и end-to-end экспериментов, оценки качества ответов и анализа retrieval-метрик.

## Цель проекта

RAG-системы часто тестируются на чистых базах знаний, однако в реальных условиях база может содержать устаревшие, дублирующиеся, повреждённые или фактически противоречивые документы.

Цель проекта — проверить, насколько разные подходы к построению RAG устойчивы к загрязнению базы знаний.

Основной исследовательский вопрос:

> Как меняется качество RAG-системы при добавлении в базу знаний семантического, контрафактического, дублирующего и структурного шума?

Проект анализирует не только итоговый ответ модели, но и поведение retrieval-этапа: удаётся ли системе найти корректные документы при наличии шумовых и противоречивых фрагментов.

## Что реализовано

- Построена векторная база знаний на основе HotpotQA.
- Реализована загрузка документов и эмбеддингов в Qdrant.
- Сгенерированы разные типы шума:
  - семантические дистракторы;
  - контрафактические фрагменты;
  - дублирующиеся и перефразированные документы;
  - структурно повреждённые тексты.
- Проведены controlled-context эксперименты.
- Проведены end-to-end эксперименты с retrieval из Qdrant.
- Реализованы и сравнены несколько RAG-подходов:
  - baseline dense RAG;
  - query expansion;
  - HyDE;
  - reranking;
  - Self-RAG-style фильтрация;
  - wide retrieval + MMR + reranking.
- Добавлено кэширование промежуточных результатов.
- Добавлена логика безопасной остановки при API-лимитах.
- Добавлена поддержка нескольких Groq API-ключей для длинных экспериментов.
- Проведена оценка качества по нескольким группам метрик.
